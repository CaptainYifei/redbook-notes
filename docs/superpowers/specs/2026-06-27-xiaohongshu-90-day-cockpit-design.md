# 小红书推荐算法账号 90 天执行驾驶舱 V1 设计

日期：2026-06-27

## 1. 背景与目标

用户要打造一个“小红书推荐算法求职与工业实践号”。已有 90 天执行计划，核心路径是：

- 先完成账号定位和主页基础设施。
- 连续发布 30 天内容验证方向。
- 根据数据筛出主线爆款模型，并在 60-90 天阶段系列化放大。

V1 的目标不是做一个完整运营后台，而是做一个每天能打开使用的本地执行工具：

> 打开 30 秒内知道今天做什么，写完能记录进度，周末能复盘方向。

## 2. 产品定位

产品名称：推荐算法小红书 90 天执行驾驶舱

形态：单页本地 Web 工具，数据保存在浏览器本地，支持导出 JSON 备份。

优先解决的问题：

- 今天该推进哪件事。
- 本周应该发哪类内容。
- 置顶三帖、账号简介、合集等启动任务是否完成。
- 选题库是否持续积累。
- 每篇笔记发布后如何根据数据复盘。

## 3. V1 范围

### 3.1 包含

1. 首页驾驶舱
   - 今日焦点任务。
   - 下一篇笔记摘要。
   - 本周发布节奏。
   - 启动清单进度。
   - 复盘入口。

2. 30 天发布计划
   - 预置前 4 周、每周 4 篇节奏。
   - 周一：求职路线。
   - 周三：深度学习基础。
   - 周五：推荐算法论文 / 工业实践。
   - 周日：个人经历 / Q&A / 工作日常。
   - 每篇内容可编辑状态：未开始、写作中、已发布、已复盘。

3. 账号基础设施任务
   - 修改账号简介。
   - 重命名合集。
   - 新增“推荐算法工程师 Offer 路线”合集。
   - 完成 3 篇置顶帖：人设帖、路线帖、论文清单帖。

4. 选题库
   - 按 4 条主线组织：推荐算法 Offer 路线、深度学习基础、推荐算法论文到工业实践、大厂算法工程师真实日常。
   - 字段包括：选题、内容线、目标用户、用户痛点、核心结论、评论钩子、状态。
   - 支持新增、编辑、筛选、状态切换。

5. 笔记模板
   - 标题公式。
   - 开头公式。
   - 正文 5 模块。
   - 结尾金句。
   - 评论区引导公式。
   - 小红书技术卡片默认规则：3:4 竖版、每张卡一个核心判断、优先结构图和工程对比。

6. 数据复盘
   - 每篇笔记可录入曝光、阅读 / 播放、点赞、收藏、评论、关注、主页访问、完读、私信数。
   - 自动计算收藏率、评论率、关注转化、完读率。
   - 根据规则给出复盘提示。

7. 本地保存与备份
   - 使用 `localStorage` 保存任务、选题、发布计划和指标数据。
   - 提供导出 JSON 备份。
   - 提供导入 JSON 恢复。

### 3.2 暂不包含

- 不接入小红书 API。
- 不自动抓取平台数据。
- 不做账号登录、云同步或多人协作。
- 不自动生成完整小红书图片。
- 不创建系统级提醒或定时自动化。
- 不做复杂图表分析；V1 只做轻量指标和判断提示。

## 4. 信息架构

V1 使用单页应用结构，顶部为固定导航：

1. 首页驾驶舱
2. 发布日历
3. 选题库
4. 笔记模板
5. 数据复盘
6. 设置 / 导出

### 4.1 首页驾驶舱

首页是默认入口，面向每天执行。

模块：

- 今日焦点：根据当前 Day 和任务状态展示最应该完成的一件事。
- 下一篇笔记：展示最近一篇未发布内容的标题、主线、核心结构和评论钩子。
- 本周发布节奏：显示本周周一 / 周三 / 周五 / 周日内容。
- 启动清单：显示简介、合集、置顶帖完成度。
- 快速入口：跳到选题库、笔记模板、数据复盘。

### 4.2 发布日历

展示前 30 天发布计划。V1 以列表或紧凑网格呈现，每条计划包括：

- Day / 周次 / 星期。
- 内容标题。
- 内容线。
- 目的：拉新、收藏、专业度、互动等。
- 状态。
- 备注。

### 4.3 选题库

选题库面向持续积累和筛选。

默认预置 4 条主线和种子选题：

- 求职路线：15 个方向。
- 深度学习基础：10 个方向。
- 推荐算法论文：15 个方向。
- 大厂工作日常：10 个方向。

每条选题都可编辑并切换状态。

### 4.4 笔记模板

笔记模板不是富文本编辑器，而是写作结构提示区。

它提供：

- 标题公式：面试会问、工作会用、求职需要、新人误解、大厂真实场景。
- 开头公式：先讲用户痛点，再解释知识点解决什么问题。
- 正文公式：问题是什么、为什么重要、传统做法问题、模型 / 概念如何解决、面试或工作怎么表达。
- 结尾公式：一句金句总结。
- 评论钩子：引导身份、卡点、下一篇选择和求职痛点。

### 4.5 数据复盘

数据复盘按笔记维度录入。

基础指标：

- 曝光。
- 阅读 / 播放。
- 点赞。
- 收藏。
- 评论。
- 新增关注。
- 主页访问。
- 完读。
- 私信数。

计算指标：

- 收藏率 = 收藏 / 阅读。
- 评论率 = 评论 / 阅读。
- 关注转化 = 新增关注 / 主页访问。
- 完读率 = 完读 / 阅读。

判断提示：

- 收藏高、评论低：内容有长期价值，但互动钩子弱。
- 评论高、收藏低：话题有讨论性，但干货不足。
- 点赞高、关注低：单篇不错，但账号承诺不清晰。
- 曝光低、互动高：选题不错，标题和封面要优化。
- 曝光高、收藏低：标题吸引人，但正文价值不够。

## 6. Interaction Design

状态切换应轻量：

- 任务：待做、进行中、完成、跳过。
- 笔记：未开始、写作中、已发布、已复盘。
- 选题：未写、已写、已发、已复盘。

核心交互：

- 勾选启动任务后首页进度立即更新。
- 修改发布计划后，首页“下一篇笔记”自动选择最近一篇未发布内容。
- 录入复盘数据后，系统自动计算指标和判断提示。
- 导出 JSON 会下载当前完整状态。
- 导入 JSON 前需要提示会覆盖当前本地数据。

## 7. Visual Direction

界面应像一个实用工作台，而不是营销落地页。

设计原则：

- 信息密度适中，优先可扫描。
- 首页第一屏直接呈现今日任务和下一篇笔记。
- 使用标签、状态、进度条、表格、筛选和小型指标卡。
- 卡片只用于任务、笔记、指标等独立信息块，不做层层嵌套。
- 颜色语义稳定：蓝色表示学习 / 基础，绿色表示选题 / 内容资产，橙色表示发布 / 决策，灰色表示流程与辅助信息。
- 小红书内容模板遵循移动端卡片规则：3:4 竖版、每张卡一个核心判断、结构图优先。

## 8. Error Handling

- 数字输入为空或非法时按 0 处理，但界面应提示用户修正。
- 分母为 0 时不计算比例，显示为 `-`。
- 导入 JSON 失败时保留现有数据并显示错误提示。
- `localStorage` 写入失败时提示用户导出当前 JSON，避免数据丢失。
- 初始化时如无本地数据，加载默认 90 天计划种子数据。

## 9. Acceptance Criteria

V1 完成后应满足：

- 打开页面后能看到今日焦点、下一篇笔记、本周节奏和启动清单。
- 可以勾选或切换任务状态，刷新页面后状态保留。
- 可以查看并修改前 30 天发布计划，刷新后保留。
- 可以新增、编辑、筛选选题。
- 可以录入至少一篇笔记的数据，并看到自动计算的收藏率、评论率、关注转化和完读率。
- 可以看到基于指标的复盘提示。
- 可以导出当前数据为 JSON，并重新导入恢复。
- 不依赖后端服务即可运行。

## 10. Implementation Notes

建议第一版直接在当前工作区新增一个独立页面或小型静态应用，优先复用现有前端风格和部署方式。

如果当前项目以静态 HTML 为主，V1 可采用：

- 一个独立 HTML 入口。
- 原生 JavaScript 管理状态。
- CSS 写在同一文件或单独样式文件中。
- `localStorage` 做本地持久化。

如后续要扩展：

- 加系统级提醒。
- 加小红书卡片图文生成器。
- 加更丰富的趋势图和内容主线对比。
- 加云同步或表格导入导出。

这些都应在 V1 跑通连续执行后再做。
