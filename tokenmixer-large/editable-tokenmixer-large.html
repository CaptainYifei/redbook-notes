02
/ Residual

Residual 形状 / 语义不对齐
→ Mixing & Reverting

问题
Token Mixing 后：
1）形状可能不对齐（H ≠ T）；
2）即使形状对齐（H = T），mixed token 的语义也已混乱，无法自然残差。
解决的效果
✓
形状对齐：
确保输出与输入在 [T, D] 上一致。
✓
语义对齐：
让 residual 发生在语义一致的空间。
✓
结构更灵活：
支持多种混合模块，如 FFN、MoE 等。
✓
深层更稳定：
每层残差更自然，信息不易混乱。

03 / Deep training

深层梯度更新不足
→ Inter-layer Residual + Auxiliary Loss

问题： 深层模型中，浅层和中间层离最终 loss 太远，
梯度变弱，block 容易学不充分。
带来的效果
↗
浅层信息
更容易传到深层
▽
梯度更易
回传到底层
⚙
低层 / 中层 block
更新更充分
✓
模型更可训练
更稳定

/ Sparse MoE

MoE 稀疏化不完整
→ Sparse-Pertoken MoE

问题：
RankMixer 的 DTSI-MoE 训练阶段仍偏 dense，
推理时 sparse，训练成本仍较高。
带来的效果
⌬
训练和推理
都更稀疏
✪
保留 per-token
独立建模优势
↗
router 学习
更容易
▣
参数容量更大，
但计算成本可控
🚀
04/06 · Sparse per-token experts for scalable MoE.

/ SwiGLU

→ Per-token SwiGLU

↗

⌬
